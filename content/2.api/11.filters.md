---
title: Filters
description: Manages audio effects for the player, including equalization, timescale, karaoke, and more.
icon: lucide:sliders
---

The `Filters` class allows you to apply audio effects to the player's output. You can use built-in presets (like "nightcore" or "bassboost") or manually configure specific filters like the equalizer or timescale.

## Properties

::api-properties{name="Filter Properties" description="Direct access to filter configuration objects."}
::api-section{title="Standard Filters"}
::
::api-schema{name="volume" type="number | undefined"}
Adjusts the player volume (0.0 to 5.0, where 1.0 is 100%).
::
::api-schema{name="equalizer" type="IEqualizerBand[] | undefined"}
Array of 15 bands (0-14) with gains from -0.25 to 1.0.
::
::api-schema{name="karaoke" type="IKaraoke | undefined"}
Karaoke settings (vocal removal).
::
::api-schema{name="timescale" type="ITimescale | undefined"}
Changes speed, pitch, and rate of the audio.
::
::api-schema{name="tremolo" type="ITremolo | undefined"}
Adds a wavering effect to the volume.
::
::api-schema{name="vibrato" type="IVibrato | undefined"}
Adds a wavering effect to the pitch.
::
::api-schema{name="rotation" type="IRotation | undefined"}
Rotates the audio around the stereo channels (8D effect).
::
::api-schema{name="distortion" type="IDistortion | undefined"}
Adds distortion/clipping to the audio.
::
::api-schema{name="channelMix" type="IChannelMix | undefined"}
Mixes left/right channels (e.g. mono).
::
::api-schema{name="lowPass" type="ILowPass | undefined"}
Low-pass filter (muffles high frequencies).
::

::api-section{title="Management"}
::
::api-schema{name="active" type="Set<string>"}
A Set of currently enabled preset/custom filter names.
::
::api-schema{name="available" type="string[]"}
List of all available built-in and custom defined filters.
::
::

## Methods

### enable(name)
::api-card{name="enable(name)" method="enable" returns="this" anchorId="enable"}
#description
Enables a pre-defined filter (builtin or custom) by name.

#default
::api-schema{name="name" type="string" required}
The name of the filter to enable (e.g., "nightcore", "bassboost-high").
::

#example
```ts
player.filters.enable("nightcore");
await player.filters.apply();
```
::

### disable(name)
::api-card{name="disable(name)" method="disable" returns="this" anchorId="disable"}
#description
Disables an active filter by name.

#default
::api-schema{name="name" type="string" required}
The name of the filter to disable.
::

#example
```ts
player.filters.disable("nightcore");
await player.filters.apply();
```
::

### clear()
::api-card{name="clear()" method="clear" returns="this" anchorId="clear"}
#description
Clears ALL active filters and resets the configuration to default.

#example
```ts
player.filters.clear();
await player.filters.apply();
```
::

### apply()
::api-card{name="apply()" method="apply" returns="Promise<Player>" anchorId="apply"}
#description
Sends the current filter configuration to the Lavalink node. You **must** call this after making changes for them to take effect.

#example
```ts
player.filters.setTimescale({ speed: 1.2 });
await player.filters.apply();
```
::

### setEqualizer(bands)
::api-card{name="setEqualizer(bands)" method="setEqualizer" returns="this" anchorId="setequalizer"}
#description
Sets custom equalizer bands.

#default
::api-schema{name="bands" type="IEqualizerBand[]" required}
Array of band objects `{ band: number, gain: number }`.
::

#example
```ts
player.filters.setEqualizer([
    { band: 0, gain: 0.2 },
    { band: 1, gain: 0.15 }
]);
await player.filters.apply();
```
::
